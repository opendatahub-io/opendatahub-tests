# TC-E2E-001: Complete job lifecycle — submit, queue, admit, run, complete

**Objective**: Verify the complete end-to-end lifecycle of a Kueue-managed evaluation job from submission through completion.

**Preconditions**:

- Kueue Operator installed on the cluster
- ClusterQueue `eval-cq` created with nominalQuota: cpu=1, memory=4Gi
- LocalQueue `eval-queue` created in test namespace mapped to `eval-cq`
- Namespace labeled with kueue.openshift.io/managed=true and evalhub.trustyai.opendatahub.io/tenant=true
- EvalHub deployed and accessible

**Test Steps**:

1. **Setup**: Install Kueue Operator (if not present), create ClusterQueue `eval-cq` (cpu=1, memory=4Gi), create LocalQueue `eval-queue`, label namespace
2. **Submit**: POST /api/v1/evaluations/jobs with queue specification (`queue.kind: "kueue"`, `queue.name: "eval-queue"`)
3. **Verify submission**: Response is HTTP 202, capture `resource.id`
4. **Verify pending state**: GET /api/v1/evaluations/jobs/{id} returns `status.state: "pending"`
5. **Verify Kubernetes Job**: A Job is created with label `kueue.x-k8s.io/queue-name: eval-queue`
6. **Verify Workload**: A Workload resource is created and eventually shows `Admitted=True`
7. **Verify running state**: Poll GET /api/v1/evaluations/jobs/{id} until `status.state: "running"`
8. **Verify pod running**: The evaluation pod is in Running state
9. **Verify completion**: Poll GET /api/v1/evaluations/jobs/{id} until `status.state: "completed"`
10. **Verify Workload finished**: Workload shows `Finished=True`
11. **Verify quota released**: LocalQueue shows admittedWorkloads decremented
12. **Teardown**: Delete the evaluation job, LocalQueue, ClusterQueue, and revert namespace labels

**Expected Results**:

- Job transitions through states: pending → running → completed
- Each state is correctly reflected in both EvalHub API and Kubernetes resources
- Workload resource transitions: QuotaReserved=False → QuotaReserved=True, Admitted=True → Finished=True
- Kubernetes Job transitions: suspend=true → suspend=false → Complete
- Quota is reserved during execution and released after completion
